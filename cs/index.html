Small Shell in C

OSU Operating Systems Project

I wrote a simplified bash-like shell in C for my Operating Systems course. It supports the built-in commands exit, cd, and status, redirection of standard input and standard output, comments, some variable expansion, and foreground and background processes, along with a few other bells and whistles. This was a neat way to get more practice with C strings, file I/O, signals, forking processes, and dealing with zombie processes.

RPG Character Manager

I'm working on a web app to help gamers create and keep track of their tabletop RPG characters. Users can search game reference materials, build characters and level them up, and manage all of their saved characters' stats and equipment in one simple interface. When complete, it'll give gamers a one-stop shop for creating and managing characters and researching game details, greatly reducing administrative time and maximizing gaming time. Currently supports Dungeons & Dragons 5e, but I'd love to add support for other RPGs at some point.

File Sharing Network Application

I wrote a file transfer program for my Networks course consisting of a file transfer server written in C and a file transfer client written in Python. The server and client establish two TCP connections: a control connection for sending/receiving commands and error messages, and a data connection for sending/receiving data such as text files. An interesting problem that came up was making sure large files were received intact, especially when the server sent data faster than the client could receive it.

CUDA Monte Carlo Simulation

In this project for my Parallel Programming course, I took a scenario involving a laser beam pointed at a circle that is randomly changing size and location, and developed a Monte Carlo simulation for it to determine the probability of the beam ricocheting off the circle and hitting a plate below. Running the hundreds of thousands of trials in this simulation was a good use for GPU parallelism! I used C++ and CUDA to pass data between the CPU and GPU, and run and time the simulation.
